# 操作手册 — 每月上报新报表

每月底拿到亚博包网商提供的报表后，照这份做就能把数据全部送进面板。

---

## 一、月度报表（8 张）

### 你会拿到哪些档

| 报表名 | 档名样式 | 用途 |
|---|---|---|
| 平台报表（经营报表） | `business-report【2026-06-01,2026-06-30】.xlsx` | 注册、首存、转化、流水概况 |
| 财务报表 | `finance-report【2026-06-01,2026-06-30】.xlsx` | 各支付渠道入款金额 |
| 推广报表 | `日期_一级_二级_xxx.csv`（或 zip） | 推广层级流水分布 |
| 代理报表 | `代理报表xxx.csv`（或 zip） | 每个代理每日 KPI |
| 会员报表 | `会员账号_注册时间_xxx.csv`（或 zip） | 每个会员当月累计 |
| TOP 报表 | `top-report【...】.xlsx`（或纯 `top.xlsx`） | 当月输赢 TOP 200 会员 |
| 游戏报表 | `游戏报表【...】.xlsx` | 每个游戏当月统计 |
| 游戏报表（场馆） | `游戏报表(场馆)【...】.xlsx` | 每个场馆当月统计 |

> **注**：TOP 报表如果叫 `top.xlsx`（没含 6 位数字月份），请改名成 `top-report 202506.xlsx`（中间要有空格 + 6 位月份），否则导入工具的正则抓不到月份。

### 操作步骤

1. 把 8 个档**全部放进** `dashboard/月度报表/`（直接复制，不需要按月份分子资料夹）。
2. 双击 `报表导入工具.bat`。
3. 看到菜单 → 输入 `1` 按 Enter。
4. 工具会问「报表文件夹路径」→ 直接 Enter（用默认 `月度报表/`）或贴完整路径。
5. 等它跑完，会显示每张表写了几笔到 BQ。

### 完成检查

跑完打开 BigQuery Console 或面板，确认这 8 张 `raw_*` 表的 `_snapshot_month` 或 `日期` 最大值 = 你这个月。

---

## 二、红利记录（CSV，按月归档）

### 你会拿到什么

包网商给的是「红利记录 第 X 部分 YYYYMMDD.zip」这类压缩档（3 个 zip 是常态，因为数据切档）。**所有 zip 都用统一密码** `1qazxcvb0401` 解开。

每个 zip 解开会有 1-3 个 CSV，档名是后台流水号（例：`20260602106329.csv`）。

### 操作步骤

1. 在 `dashboard/红利记录/` 底下建一个该月资料夹：例 6 月 = `红利记录/202506/`。
2. 解压所有红利 zip（密码 `1qazxcvb0401`）。
3. 把所有 CSV 全部放进 `红利记录/202506/`。
4. 双击 `报表导入工具.bat`，输入 `2` 按 Enter。
5. 工具会自动扫描所有月份资料夹（不只新月份），重新写整张 `raw_bonus_report` 表（这是「全表覆盖」式，不是追加）。

### 重要提醒 ⚠️

红利记录是「全表覆盖」（WRITE_TRUNCATE）写入 BQ。**所以每次跑都必须保留所有历史月份的 CSV**，不能把旧月份资料夹删掉，否则历史红利数据会消失。

---

## 三、客服对话（XLSX，按月归档）

### 你会拿到什么

包网商每月给一份 xlsx，里面有 32 个 sheet（1 张「总表」+ 31 天 daily 分页），档名样式：`5月客服对话.xlsx`、`6月客服对话.xlsx`。

### 操作步骤

1. 在 `dashboard/客服对话/` 底下建一个该月资料夹：例 6 月 = `客服对话/202506/`。
2. 把 xlsx 放进去。
3. 双击 `报表导入工具.bat`，输入 `6` 按 Enter。
4. 工具会自动扫所有月份资料夹 + 全部 xlsx + 全部 sheets，写整张 `raw_cs_conversations` 表（同样是「全表覆盖」式）。
5. 工具会**自动跑「不满意主因抽取」**——每笔差评对话会自动标主因，存到 BQ 的 `_extracted_issue` 栏。

### 同样要保留历史

跟红利记录一样，每次跑要保留所有月份资料夹。

---

## 四、如果报表抓不到（少了某张表）

最常见两个坑：

### 坑 1：TOP 报表文件名 `top.xlsx` 没含 6 位数字

→ 改名 `top-report 202506.xlsx`（中间空格 + 6 位月份）再跑。

### 坑 2：红利 zip 没解到对应资料夹

→ 检查 `红利记录/202506/` 里面是不是 csv，不是 zip。zip 要先解。

---

## 五、跑完一定做一次检查

打开云端面板（`miru-ops-dashboard.streamlit.app`） → 检查这几页有没有显示新月份：

- 经营总览 — 看「快速切换月份」下拉有没有新月份
- 红利分析 — 同上
- 客服分析 — 看「对话总数」是否 = 总笔数 + 「平均首次响应」非 0
- 代理 × 会员 明细 — 拉「代理总览」看有没有新增代理

任何一个看不到 = 该张表没 import 成功 → 回头跑相应 import 选项。

---

## 六、不小心跑两次会重复吗

**不会**——工具有 idempotent 保护：

- 每张表写入前，工具先去 BQ 查「同档名 (`_source_file`) 是否已写过」
- 如果是 → **跳过 + 印 ⚠️ 警告**，避免重复
- 如果不是 → 正常 import

所以你跑 2 次 / 3 次同月份，数据都不会变 × 2 / × 3，只会看到一堆「已有 X 笔来自 'xxx.xlsx' → 跳过 (避免重复写入)」的讯息。

### 例外情况：包网商修正后给你同档名但内容不同的报表

工具会**跳过**（因为它只看档名，不看内容）。这种 case 通知工程师用 BigQuery Console 手动 DELETE 旧资料后再跑导入。

### 红利记录 + 客服对话

这两张是「全表覆盖」(TRUNCATE) 写入，每次跑都重写整张表，所以**重复跑也不会重复**（但要记得保留所有月份历史资料夹，不然历史数据会丢）。

---

如果导入工具回报「写入失败」，把完整错误讯息发给工程师协助。
