# 运营数据面板 — 首读

这个资料夹是整套「运营数据面板」的完整安装包，含：
1. 云端面板（`dashboard.py`）— 在 `miru-ops-dashboard.streamlit.app` 上 24 小时运转
2. 报表导入工具（`import_tool.py`）— 把每月新报表上传到 BigQuery
3. 启动 BAT — 双击就能用
4. 数据上传资料夹（`月度报表/`、`红利记录/`、`客服对话/`）

> **看完这份之后** 请再依序看 `2_操作手册_每月上报.md` 跟 `3_分页功能说明.md`。

## 一、整套架构是怎么运作的

```
[亚博包网商 / 后台 / 客服系统]
         │
         │ 每月底拿到 8 类 Excel / CSV / Zip 报表
         ▼
[本机这个资料夹 dashboard/]
         │
         │ 把报表放进对应的子资料夹
         ▼
[报表导入工具 import_tool.py]
         │
         │ 清洗 + 上传 BigQuery
         ▼
[Google BigQuery] ← 11 张原始资料表
         │
         │ Streamlit 面板每 5 分钟自动刷一次
         ▼
[云端面板] miru-ops-dashboard.streamlit.app
         │
         ▼
[Miru 看面板 → 做决策]
```

## 二、每月你只要做 3 件事

### 1. 报表来了 → 放对资料夹

| 报表类型 | 放哪 |
|---|---|
| 8 张月度报表（平台、财务、游戏、会员、代理、TOP 等） | `月度报表/` |
| 红利记录（CSV，按月分压缩档） | `红利记录/202506/` 这种命名 |
| 客服对话 xlsx | `客服对话/202506/` 这种命名 |

> 月份资料夹命名一律 6 位数字 `YYYYMM`（例如 6 月 = `202506`）。

### 2. 双击「报表导入工具.bat」选 1 / 2 / 6

- 选 **1** → 导入 8 张月度报表
- 选 **2** → 导入红利记录
- 选 **6** → 导入客服对话

每个选项都会显示 import 进度 + 最后写入 BQ 几笔。

### 3. 打开云端面板（或双击「启动运营面板.bat」开本机版）

5 分钟内 BQ 资料 cache 过期，新月份资料就出现在面板上。

## 三、出问题最快的排查路径

| 现象 | 原因多半是 | 解法 |
|---|---|---|
| 面板看不到 5 月（或新月份）数据 | 该张表没 import 进 BQ | 重跑 `报表导入工具.bat` |
| 某个分页有红框报错 | 代码版本不同步 / BQ 表结构变 | 看错误讯息 + 检查云端 GitHub 是否最新 |
| 导入工具说「资料夹不存在」 | 子资料夹少了 / 命名错 | 比对 `4_资料夹结构说明.md` |
| 满意度 / 接待客服全 0 | 字段不在 dashboard 白名单 | 联系工程师改 `TEXT_COLUMNS` |
